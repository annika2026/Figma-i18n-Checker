# 缓存功能快速使用指南

## 功能特点

✅ **自动启用**：无需额外配置，在 `validateI18nTexts` 方法中自动工作  
✅ **智能过滤**：自动跳过重复的 key+language+text 组合  
✅ **自动清空**：每次检测开始时自动清空上一次缓存  
✅ **性能提升**：避免重复检测，提高检测效率  
✅ **统计透明**：提供详细的命中率和缓存统计信息  

## 快速开始

缓存功能已经内置，无需任何额外配置：

```javascript
const TextWidthChecker = require('./textWidthChecker');
const checker = new TextWidthChecker();

// 直接使用，缓存自动工作
const results = checker.validateI18nTexts(textNodes, translations);

// 查看缓存统计
console.log(results.cacheStats);
// 输出：
// {
//   total: 9,           // 总请求次数
//   hits: 3,            // 缓存命中（跳过检测）
//   misses: 6,          // 实际检测次数
//   cacheSize: 6,       // 唯一文本数量
//   hitRate: "33.33%"   // 命中率
// }
```

## 工作原理

### 缓存生命周期

```
新检测开始
    ↓
清空上次缓存  ← [重要] 确保每次检测完整
    ↓
检测文本节点
    ↓
遇到新文本 → 执行检测 → 添加到缓存 → 记录结果
    ↓
遇到重复文本 → 跳过检测 → 不记录结果 → 节省时间
    ↓
检测完成 → 输出缓存统计
```

### 缓存键规则

缓存键由三部分组成：**key + language + text**

```javascript
// 这些会被缓存：
'button.submit|zh-CN|提交'     → 首次检测
'button.submit|zh-CN|提交'     → 命中缓存 ⚡
'button.submit|zh-CN|提交'     → 命中缓存 ⚡

// 这些不会命中缓存（key 不同）：
'button.submit|zh-CN|提交'     → 首次检测
'button.cancel|zh-CN|提交'     → 首次检测（key 不同）

// 这些不会命中缓存（language 不同）：
'button.submit|zh-CN|提交'     → 首次检测
'button.submit|en-US|提交'     → 首次检测（language 不同）
```

## 常见问题

### Q1: 缓存会不会导致某些文本没被检测？
**A**: 不会。每次检测开始时会自动清空缓存，确保每次检测都是完整的。缓存只在单次检测内跨节点生效。

### Q2: 被缓存的文本会记录到结果中吗？
**A**: 不会。命中缓存的文本会被跳过，不会记录到 `valid` 或 `overflow` 数组中，避免重复记录。

### Q3: 如何知道缓存的效果？
**A**: 检查返回结果中的 `cacheStats` 字段，包含命中率等详细统计信息。

### Q4: 可以手动清空缓存吗？
**A**: 可以，调用 `checker.clearCache()`，但通常不需要，系统会自动管理。

### Q5: 缓存对性能提升有多大？
**A**: 取决于重复节点的数量。如果有 30% 的重复节点，性能可提升约 30%。
